# GET /dox/ip — докс по IP
Query:
- ip — обязательный, ну типо это IP, подходит и IPv4 и IPv6, через запятую можно писать несколько IP (IPv6 не надо экраниривать в квадратные скобки)
- lang — предпочитаемый язык, названия стран/городов/континентов будут отображатся именно на этом языке если есть перевод, доступные переводы: ["de","en","es","fa","fr","ja","ko","ru","pt-BR","zh-CN"], по дефолту "ru"


Ответ 200, SUCCESS_IP_DOX, для query ip=178.141.246.53,127.0.0.1,192.168.1.1,254.254.254.254,<какой-то IP которого нет в базе>,arbuz,fe80::ccfe:c4fe:c12b:f216,224.0.0.1:
```jsonc
[
  {"result": "success", "continent_code": "EU", "continent": "Европа", "is_in_european_union": false, "country_code": "RU","country": "Россия", "subdivisions": ["Kirov Oblast"], "city": "Kirov", "latitude": 58.603, "longitude": 49.6679, "ASN": 8359, "ASO": "MTS PJSC"},
  {"result": "localhost"}, // На 127.0.0.1
  {"result": "private ip"}, // На 192.168.1.1 так как это приватный IP
  {"result": "ip not exists"}, // На 254.254.254.254 так как такой IP не использован в интернете
  {"result": "ip not found"}, // На айпи которого нет в базе
  {"result": "invalid ip format"}, // На arbuz так как это невалидный IP
  {"result": "link local"}, // На fe80::ccfe:c4fe:c12b:f216, так как это локальный интерфейсный адрес
  {"result": "multicast"} // 224.0.0.1 так как это multicast адрес (в эту же категонию входит broadcast типо 255.255.255.255)

```

Примечания:
- Информация по IP в итоговом массиве отображается ровно в том порядке в котором находились IP в поле ip
- Больше result-ов не будет, всё что выше это все возможные result, их можно объявить как enum
